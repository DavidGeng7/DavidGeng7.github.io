Overview

Object-Oriented Programming (OOP) is a type of programming that is centred around creating objects with attributes and controlling them through methods.

An object is created from a class, which serves as a blueprint for the object. A class contains variables, which define the characteristics of the object created, and methods, which define the action of the objects created.

All classes contain a special method called a constructor, which the class uses to create the object and initialize the fields of the class. The constructor always has the same name as the class and without a return type. If a class has no explicit constructor, Java provides a default empty no-argument constructor.

An example Dog class: 

public class Dog
{
    private String name;
    private int age;
    private String owner;

    public Dog()
    {
        name = "Bob";
        age = 2;
        owner = "Dave";
    }

    public Dog(String n, int a, String o)
    {
        name = n;
        age = a;
        owner = o;
    }

    public void bark()
    {
    	System.out.println(“woof woof”);
    }
}

The dog class has 3 fields, the dog’s name, the age of the dog, and the owner’s name.
The class has one method called bark(), which prints “woof woof” when called.
There are two constructors, one with arguments, and one without.

A Dog object can be created in the main method: 

public static void main (String [] args)
{
    Dog dog1 = new Dog(“John”, 4, “Kev”);
    dog1.bark();
}

This creates a new instance/object of the Dog class and makes it bark. One can create multiple instances of the same class with different arguments.

Encapsulation

You might ask, why do we set the attributes of a class as private instead of public? What difference does it make? This is where the idea of encapsulation comes in. Encapsulation is a core concept in OOP which basically states that a programmer should make sure that one can only access a minimal amount of information in the class instead of the entire class.

To achieve this in Java, one can use access modifiers: public and private to control the access level.

Only methods inside the class can access the private variables of a class, while public methods can be accessed outside of the class. Going back to the Dog class, name, age, and owner are all private, meaning they can only be accessed by methods inside the class. While the bark method is public, which is why we can access it in the main method.

So how to access private variables of a class then? One should employ get and set methods. The get method will return the variable value, and the set method will set the value of the variable.

An example of get and set methods: 

public class Dog
{
    private String name;
    private int age;
    private String owner;
        
    public Dog()
    {
        name = “Bob”;
        age = 2;
         owner = “Dave”;
        
    public Dog(String n, int a, String o)
    {
        name = n;
        age = a;
        owner = o;
    }
    
    public void bark()
    {
        System.out.println(“woof woof”);
    }

    public String getName()
    {
        return name;
    }

    public String setName(String n)
    {
        name = n;
    }
}

The getName() will return the Dog's name when called, and the setName will set the name of the dog to the one passed in the parameter.

In conclusion, Object-Oriented Programming is a form of programming that focuses on the use of objects or instances of classes.
A class is a blueprint for building objects and include:

  • Variables to define the attributes of the object
  • Methods to define the behaviour of the object


One should always consider encapsulation in OOP as it restricts access to parts of the class. Use get and set methods to read and/or write data into the class when needed.
