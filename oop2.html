Overview

This tutorial will teach you more advanced OOP concepts in Java, namely inheritance and polymorphism. If you have not read the basic OOP tutorial, be sure to read it first as you need the basic concept before moving onto a higher level.

Letâ€™s suppose that you have two very similar classes, the car class and the raceCar class, both with similar attributes and methods. But you are lazy and do not wish to write the full code for both classes. You can use the concept of inheritance to help you.


Consider the following Car and RaceCar class

public class Car
{
    private String color;
    private int gas;

    public Car() {
        color = gray;
        gas = 100;
    }

    public void drive (int km)
    {
        gas -= km/3;
    }
}
    
public class RaceCar
{
    private String color;
    private int gas;
    private int topSpeed;

    public RaceCar()
    {
        color = gray;
        gas = 100;
        topSpeed = 300;
    }

    public void drive (int km)
    {
        gas -= km/3;
    }

    public void race ()
    {
        gas -= km;
    }
}

As you can see, the Car and RaceCar class share many similarities and repeated code which can be negated. You can also see that RaceCar really is just a type of Car.


Although there are still some differences between them, they are largely the same thing. We can use the concept of inheritance to negate the extra code. 

//Same Car class
public RaceCar extends Car
{
    private int topSpeed;

    public RaceCar()
    {
        super();
        topSpeed = 300;
    }

    public void race ()
    {
        gas -= km;
    }
}

What we did here is that we made Car a superclass and RaceCar a subclass. The extends keyword marks the RaceCar as a subclass of the Car superclass. This means that RaceCar will inherit all the attributes (variables) and methods of the Car class without writing the same code again.

The super() calls the super constructor, which essentially creates a Car object within the RaceCar class which allows the subclass to inherit attributes and use the method of the superclass.

The access modifiers are still enforced even if two classes are linked by inheritance, so make sure to use get and set methods to access variables when needed.

Note that all Java objects have a common ancestor, they are all subclasses of the Object class in java.


Suppose that we create another subclass of the Car class, OldCar.


public class OldCar extends Car 
{
    public OldCar() {
        super();
    }

    public void drive (int km)
    {
        gas -= km*2;
    }
}

An old car uses a lot more fuel to drive, so when an OldCar object is declared and the drive method is called, the drive method in the class will override the drive method of the super class. This means that it will always use km*2 gas instead of km/3 gas when the drive method is run for the OldCar object. Java will always default to the method closest associated with the object type.


Polymorphism

Polymorphism is a concept in Java that basically ensures that a correct method is called for an object of a specific type, even if we reference a subclass to a superclass. This ties into the idea of method overriding which makes sure that the method that is most associated with the object is called.
