Overview

This tutorial will cover how to search for an element in an array and how to sort said array. Be sure to learn about array first in the array tutorial first before you start this tutorial.


Searching

There are two main ways to search for a value in an array, sequential search and binary search.


The most simple way to search is sequential search, where one basically loop through the entire array and compare each element with the key, or the value search for. The time complexity of this algorithm is O(n), meaning that the time it takes to loop n times increase linearly.

public static int sequentialSearch(int [] arr, int key)
{
    for(int i = 0; i < arr.length; i++)
    {
        if(key == arr[i])
            return i;
    }
    return -1;
}

The next way to search is binary search. The idea behind binary search is that if the middle value of the array or subarray is larger or smaller than the value searched, we can discard half of the current array. Repeat until there is only one value left. Downside to the search is that it requires the array to be sorted. The time complexity of the search is O(log n), which excels at large data sets as time grows slower as n grows.

public static int binarySearch(int [] arr, int key)
{
    int left = 0;
    int right = arr.length - 1;
    while(left <= right)
    {
        int mid = (left + right) / 2;
        if(arr[mid] == key)
            return mid;
        else if (key < arr[mid])
            right = mid - 1;
        else 
            left = mid + 1;
    }
    return -1;
}

Sorting

Now we will cover two sorting algorithms.


The first algorithm is selection sort, which finds the maximum value among the first nth elements. It is then swapped with the value at the current index and then the counter is decreased by 1. These steps are repeated until the length of n is equal to 1. The sort has a time complexity of O(n^2), this means that for each time the outer loop executes (which executes n times), the inner loop executes n times. This means that the sort have trouble dealing with larger data sets as the time increases quadratically.

public static int insertionSort(int [] arr)
{
    for(int i = 0; i < arr.length; i++)
    {
        int temp = arr[i];
        int index = i - 1;

        while(temp > 0 && arr[index] > temp)
        {
            arr[index] = arr[index-1];
            index--;
